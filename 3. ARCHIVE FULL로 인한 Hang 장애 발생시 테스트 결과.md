## 순서
1. 볼륨 생성 및 /arch 경로 생성
2. log_archive_dest_2 경로 추가
3. 대량 DML 발생하여 /arch FULL 만들기
4. /arch 공간을 확보한 후 다른 세션에서 alter system switch logfile 발생

## 1. 볼륨 생성 및 /arch 경로 생성

- 볼륨 : 저장 공간을 관리하는 논리적인 단위 
- Archive Full : 아카이브 로그 파일이 저장되는 디스크 공간이 가득 찬 상태

## 2. log_archive_dest_2 경로 추가
```bash
SYS @ orcl > alter system set log_archive_dest_2='LOCATION=/arch';

System altered.

SYS @ orcl > show parameter archive
log_archive_dest_1                   string      LOCATION=/home/oracle/arch
log_archive_dest_2                   string      LOCATION=/arch
# 변경 후 아카이브 파라미터
```

## 3. 대량 DML 발생하여 /arch FULL 만들기
```bash
SYS @ orcl >
-- 1. 테스트용 테이블 생성
CREATE TABLE hang_test (
    id NUMBER,
    dummy_data CHAR(2000) -- 한 행당 2KB 정도 차지하게 설정
);

-- 2. 대량 DML 발생 (PL/SQL)
BEGIN
    FOR i IN 1..500000 LOOP -- 50만 번 반복 (약 1GB 이상의 리두 발생)
        INSERT INTO hang_test VALUES (i, 'A');
        IF MOD(i, 1000) = 0 THEN
            COMMIT;
        END IF;
    END LOOP;
    COMMIT;
END;
/
```
