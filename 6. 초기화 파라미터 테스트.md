# 3-2. spfile 만 존재시, 삭제 되었을 때 복구 방법(11g + ASM)

순서
 3.2-1 spfile, pfile 모두 삭제
 3.2-2 memory를 이용하여 spfile 생성
 3.2-3 spfile명 변경 및 startup

## 3.2-1 spfile, pfile 모두 삭제
- spfile은 asmcmd에서 삭제한다
```bash
SQL> show parameter spfile

NAME                                 TYPE        VALUE
------------------------------------ ----------- ------------------------------
spfile                               string      +DATA/orcl/spfileorcl.ora

ASMCMD> pwd
+data/ORCL
ASMCMD> ls
CONTROLFILE/
DATAFILE/
ONLINELOG/
PARAMETERFILE/
TEMPFILE/
spfileorcl.ora
ASMCMD> rm spfileorcl.ora
```

```bash
[orcl:dbs]$ pwd
/u01/app/oracle/product/11.2.0/dbhome_1/dbs
[orcl:dbs]$ rm initorcl.ora
```
## 3.2-2 memory를 이용하여 spfile 생성
```bash
SQL> create spfile from memory;

File created.
```

##  3.2-3 spfile명 변경 및 startup
```bash
SQL> startup force

Total System Global Area  481259520 bytes
Fixed Size                  1337352 bytes
Variable Size             331352056 bytes
Database Buffers          142606336 bytes
Redo Buffers                5963776 bytes
Database mounted.
Database opened.
SQL> show parameter spfile;

NAME                                 TYPE        VALUE
------------------------------------ ----------- ------------------------------
spfile                               string      /u01/app/oracle/product/11.2.0
                                                 /dbhome_1/dbs/spfileorcl.ora
```

### spfile을 다시 asm 디스크에 저장하고 싶다면 ASM 경로를 지정한다
```bash
create spfile='+DATA/orcl/spfileorcl.ora' from memory;
```


## 4. pfile과 spfile을 각각 사용하여 controlfile 위치 변경 후 DB 재기동 Test
### 4.1. spfile 을 사용하여 controlfile 위치 변경 후 DB 재기동 Test
 순서
 4.1-1 control_files parameter 수정(3개에서 2개로 수정)
 4.1-2 DB 재기동

 ##  4.1-1 control_files parameter 수정(3개에서 2개로 수정)
 ```bash
SYS @ orcl > show parameter control_files

NAME                                 TYPE        VALUE
------------------------------------ ----------- ------------------------------
control_files                        string      /u01/app/oracle/oradata/ORCL/c
                                                 ontrol01.ctl, /u01/app/oracle/
                                                 oradata/ORCL/control02.ctl, /u
                                                 01/app/oracle/oradata/ORCL/con
                                                 trol03.ctl

SYS @ orcl > alter system set control_files='/u01/app/oracle/oradata/ORCL/control01.ctl',
  2  '/u01/app/oracle/oradata/ORCL/control02.ctl';
alter system set control_files='/u01/app/oracle/oradata/ORCL/control01.ctl',
                 *
ERROR at line 1:
ORA-02095: specified initialization parameter cannot be modified
```
- 동적으로 변경이 불가능하다
```bash
SYS @ orcl > alter system set control_files='/u01/app/oracle/oradata/ORCL/control01.ctl',
  2  '/u01/app/oracle/oradata/ORCL/control02.ctl' scope=spfile;

System altered.
```
